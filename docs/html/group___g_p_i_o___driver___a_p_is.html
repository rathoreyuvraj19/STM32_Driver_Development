void gpio_irq_config ( GPIOx_RegDef_t * pGPIOx,
uint8_t pin,
uint8_t trigger,
uint8_t irq_priority )

Configure a GPIO pin as an external interrupt (EXTI).

This API configures the selected GPIO pin for external interrupt usage by:

  • Mapping the GPIO port to the correct EXTI line using SYSCFG
  • Configuring the trigger type (rising/falling/both)
  • Enabling the interrupt line in EXTI
  • Setting the NVIC priority for the corresponding EXTI IRQ
Parameters
pGPIOxPointer to the GPIO port (GPIOA, GPIOB, ...) GPIO Port Base Address Macros
pinGPIO pin number (0–15) GPIO Pin Number Macros
triggerInterrupt trigger selection Interrupt Trigger Type Macros
irq_priorityNVIC priority for the EXTI interrupt (0 = highest)

Sequence of steps in gpio_irq_config() API:

  1. Program SYSCFG
  2. Program EXTI
  3. Program Trigger mode
  4. Clear pending interrupt
  5. Unmask EXTI line
  6. Sets NVIC priority
  1. Enable SYSCFG Peripheral Clock

The EXTI configuration registers (EXTICR1–4) reside inside SYSCFG.

We must enable its clock before accessing them.

Pointer to access SYSCFG register block

  1. Determine which EXTICR register to use

Each EXTICR register configures 4 pins: EXTICR1 → EXTI0..3 EXTICR2 → EXTI4..7 EXTICR3 → EXTI8..11 EXTICR4 → EXTI12..15

Index is pin / 4: pin 0–3 → 0 pin 4–7 → 1 pin 8–11 → 2

pin 12–15 → 3

  1. Determine GPIO port code for mapping

SYSCFG_EXTICR expects: 0x0 = Port A 0x1 = Port B 0x2 = Port C ...

0x8 = Port I

Invalid GPIO pointer

  1. Configure the correct SYSCFG_EXTICR register

Each pin gets 4 bits inside its EXTICR register: Bits = 4 × (pin % 4)

Steps:

  • Clear the 4-bit field for the selected EXTI line

- Write GPIO port code to select the source input

Configure EXTI0–3

Configure EXTI4–7

Configure EXTI8–11

Configure EXTI12–15

  1. Setting rising/falling trigger or both

4 Clearing the pending interrupt. A interrupt is clearing by writing 1 to EXTI_PR Register

  1. After Clearing the pending bit. Now unmask the interrupt.
  1. Finally, configure the NVIC interrupt priority

void gpio_irq_control ( uint8_t pin,
uint8_t en_di )

Enable or disable NVIC interrupt for a GPIO EXTI line.

Enable/disable NVIC interrupt for a GPIO EXTI line.

Parameters
pinGPIO pin number (0–15), used to determine EXTI line.
en_diENABLE or DISABLE.
Note
This function only updates NVIC enable registers. It does NOT configure EXTI, SYSCFG, or trigger polarity.
Note
This API only controls EXTI GPIO interrupts (EXTI0–EXTI15).

Steps to ENABLE a GPIO EXTI interrupt:

  1. Unmask the EXTI line in EXTI_IMR
  2. Enable the corresponding NVIC interrupt via NVIC_ISER
  3. (Optional but recommended) Clear any pending EXTI flag using gpio_irq_clear() Try to follow the sequence, to avoid race condition

Steps to DISABLE a GPIO EXTI interrupt:

  1. Disable the corresponding NVIC interrupt via NVIC_ICER
  2. Mask the EXTI line in EXTI_IMR
  3. Clear any pending EXTI flag using gpio_irq_clear() Try to follow the sequence, to avoid race around condition
Note
Only EXTI->PR must be written to clear pending interrupts. Writing 1 to EXTI_PR clears both the EXTI and NVIC pending state. No need to write to NVIC->ICPR for EXTI lines.
